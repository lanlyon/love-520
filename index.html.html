托管和分享步骤：

  1. 保存网页文件：
    • 复制上面的代码，粘贴到一个文本编辑器（如记事本或VS Code）。
    • 保存文件，命名为index.html。
  2. 托管网页：
    • 使用免费托管服务（推荐简单方法）：
      • GitHub Pages：
        1. 在GitHub上创建一个新仓库（例如love-520）。
        2. 将index.html上传到仓库根目录。
        3. 转到仓库的“Settings” > “Pages”，选择“main”分支，点击“Save”。
        4. 几分钟后，GitHub会生成一个链接，例如：https://你的用户名.github.io/love-520/。
      • Vercel：
        1. 访问vercel.com，注册并登录。
        2. 创建新项目，选择“Import Git Repository”或直接上传index.html。
        3. 部署后，Vercel会提供一个URL（例如https://love-520.vercel.app）。
      • Netlify：
        1. 访问netlify.com，注册并登录。
        2. 拖放index.html到Netlify的“Sites”页面。
        3. 部署后，获取一个URL（例如https://520-love.netlify.app）。
    • 这些服务都免费且简单，适合快速分享。
  3. 分享链接：
    • 部署成功后，复制托管服务提供的URL。
    • 通过微信、QQ、短信或X等平台将链接发送给对方。
    • 对方点击链接即可在浏览器中查看你的表白网页。

注意事项：

  • 确保托管服务生成的链接以https://开头，保证安全性。
  • 测试链接是否能在手机和电脑上正常打开。

如何在GitHub上创建新仓库

步骤

  1. 登录GitHub：

    • 访问 github.com，使用你的账号登录。

  2. 进入创建仓库页面：

    • 点击右上角的“+”图标，选择“New repository”。

    • 或者，在GitHub主页左侧点击“Your repositories”，然后点击绿色按钮“New”。

  3. 填写仓库信息：

    • Repository name：输入仓库名称（例如love-520）。

    • Description（可选）：简要描述仓库用途。

    • Public/Private：选择“Public”（公开，任何人可见）或“Private”（仅限授权用户）。

    • Initialize this repository with a README（可选）：勾选以创建初始README文件。

    • 其他选项（如.gitignore或license）可根据需要选择。

  4. 创建仓库：

    • 点击绿色按钮“Create repository”。

    • 仓库创建完成后，你将被重定向到仓库主页。

  5. 上传文件（如index.html）：

    • 在仓库主页，点击“Add file” > “Upload files”。

    • 拖放文件（例如index.html）或选择文件上传。

    • 输入提交信息（例如“Add 520 love webpage”），点击“Commit changes”。

注意事项

  • 确保仓库名称唯一且简洁。

  • 如果计划使用GitHub Pages托管网页，建议仓库命名为用户名.github.io（用于个人主页）或保持简单。

  • 上传文件后，检查文件是否正确显示在仓库中。

下一步

  • 若要托管网页，参考GitHub Pages设置：

    1. 进入仓库的“Settings” > “Pages”。

    2. 在“Branch”下选择main分支，点击“Save”。

    3. 几分钟后，访问生成的链接（例如https://用户名.github.io/仓库名/）。
